Portfolio Explorer Specification
================================

Purpose and Runtime
===================

Portfolio Explorer is a dependency-free static browser UI for portfolio.json files
created by the aggregator and portfolios.json files created by the collector. The
user opens explorer.html directly and chooses or drops exactly one JSON file. Data
remains local to the browser tab. No server, package manager, build step, or
third-party library is required.


Input Contract
==============

Only versioned JSON is supported; CSV is not supported.

A single-snapshot portfolio document must contain:

- schemaVersion 3 and a valid ISO YYYY-MM-DD portfolio date.
- configuration with a non-empty, unique account_columns array and asset metadata.
- exchangeRates keyed by currency with positive rateToCad values.
- yields keyed by asset.
- supplementalAssets for cash and real-estate metadata.
- holdings with unique asset identifiers and account-value objects.

Holding account keys must occur in configuration.account_columns. Missing accounts
are treated as zero. Every holding must resolve to exactly one metadata record from
configuration.assets or supplementalAssets. Duplicate metadata is invalid.
Every metadata record used by a holding supplies an allowed currency, and every
used currency must have an exchange rate.

Yield percent may be numeric or null. Market URLs are derived from providerSymbol.
Supplemental URLs must be blank or absolute HTTP/HTTPS URLs.

Malformed JSON, unsupported schema versions, invalid dates, missing metadata/rates, duplicate
assets/accounts, invalid numbers, unknown accounts, and unsafe URLs produce a visible
error instead of a partial dashboard.

A collection document must contain schemaVersion 1, kind portfolioCollection, and
a non-empty portfolios array of complete schema-version-3 portfolio documents.
Collection snapshot dates must be strictly chronological and unique.


Dynamic Calculations
====================

All values are calculated for the currently selected accounts:

    Total = sum of selected native-currency account values
    Total CAD = Total * exchangeRates[currency].rateToCad
    % Holding = Total CAD / selected portfolio Total CAD * 100
    Projected Annual Income = Total CAD * Yield / 100

Unknown yield produces unknown projected income. Grouped Total CAD and income are
sums of member assets. Grouped yield is CAD-value-weighted among assets with known
yields. Grouped holding percentage is recalculated against the selected portfolio.


Interaction
===========

- Loading portfolio.json opens Snapshot view.
- Loading portfolios.json opens History view by default.
- When portfolios.json is loaded, the app header shows one context dropdown
  containing History view followed by every embedded portfolio date.
- Embedded portfolio dates are sorted from most recent to oldest.
- Choosing a date jumps to Snapshot view for that date. Choosing History view
  switches back to the history chart without reloading the file.
- All configured accounts are selected initially.
- Select All and Clear All controls update account selection.
- Grouping choices are None, Account, Type, Market, Sector, Risk, and Currency.
- None shows assets in the table and groups the pie chart by Sector.
- Other grouping choices apply to both table and chart.
- Table headings support ascending/descending sorting; Shift-click creates a
  multi-column sort.
- Table and pie chart are visible together and stack responsively when narrow.
- The read-only FX table lists rates embedded in portfolio.json.
- Asset labels link to derived or supplemental URLs when available.


History View
============

- Measure choices are Portfolio Value and Projected Annual Income.
- Stack choices are Account and every discovered classification dimension.
- Time Frame choices are YTD, 3M, 6M, 1Y, 2Y, 5Y, Full date range, and Custom.
- From and To date controls are visible only when Custom is selected and restrict
  the x-axis inclusively.
- The chart is a filled stacked-area SVG with a total outline.
- Straight segments connect actual portfolio observations on a time-scaled x-axis.
- Hovering shows the nearest snapshot, total, category values, and income coverage.
- The legend appears to the right of the chart on wide screens and stacks below on
  narrow screens.
- The legend and tooltip show value and percent of total.
- History chart colors are assigned by visible series order, not by category hash,
  using a stable 36-color graph palette that does not change with UI color scheme.
  Visible categories do not collide unless the palette is exhausted.
- The history chart keeps a useful minimum height; short viewports may scroll
  instead of squeezing the graph.


Table and Totals
================

The asset table displays Asset, Type, Market, Sector, Risk, Currency, dynamically
calculated Total, Total CAD, % Holding, Yield, Projected Income, and configured
accounts. Yield appears immediately before Projected Income.

The sticky TOTAL footer is excluded from sorting. Each selected account column shows
its CAD-converted total. Total CAD shows the combined selected value. Weighted Yield,
% Holding, and Projected Income are recalculated dynamically. Native Total remains
blank when it would mix currencies.


Formatting and Layout
=====================

- Monetary values display as whole dollars with separators.
- Percentages use at most two decimal places.
- FX rates retain useful precision.
- Dark mode is the default; light mode is available.
- Forest, Ocean, Aubergine, and Amber color schemes are independent of theme.
- Graph colors use a stable categorical palette and do not change with the UI
  color scheme.
- Theme and color scheme persist when browser storage is available.
- The dashboard expands to browser width and fits desktop browser height.
- Overflow scrolls inside the relevant panel; narrow layouts use an internally
  scrolling main region.
- The snapshot analysis grid keeps a useful minimum height; short viewports may
  scroll instead of squeezing the table and chart.
- The front page has a top-right Choose portfolio JSON action and an inline
  choose-file link in the drag-and-drop hint.
- The front page art shows a rising time-series chart.
- The launch panel disappears after loading; selecting the loaded filename opens
  another portfolio file.


Verification
============

The pure JavaScript model is tested with:

    node --test tests\model.test.js tests\collection-model.test.js

The browser application is split into:

- explorer.js for file loading, theme, collection routing, and view switching.
- snapshot-view.js for the table, pie chart, snapshot controls, and snapshot
  account filtering.
- history-view.js for time-frame controls, history account filtering, and the
  stacked area chart.
- model.js and collection-model.js for pure data validation and calculations.

Explorer must remain usable when opened directly as a local file.
